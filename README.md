# Ubuntu Internal DNS & Mirror Setup

راهنمای ساده برای تنظیم **DNS داخلی** و **مخازن داخلی Ubuntu** جهت افزایش سرعت آپدیت و پایداری اتصال در شبکه‌هایی که محدودیت یا اختلال در اینترنت بین‌الملل دارند.

این روش به شما کمک می‌کند تا با استفاده از **DNS الوند نتورک** و **مخازن داخلی Ubuntu**، عملیات نصب و به‌روزرسانی پکیج‌ها را سریع‌تر و پایدارتر انجام دهید.

---

## مزایا

• افزایش سرعت `apt update` و نصب پکیج‌ها  
• کاهش وابستگی به اینترنت بین‌الملل  
• پایداری بیشتر در شبکه‌های محدود  
• مناسب برای سرورهای داخل ایران  

---

## پیش‌نیازها

قبل از شروع مطمئن شوید:

• سیستم عامل Ubuntu نصب شده باشد  
• دسترسی `root` یا `sudo` داشته باشید  
• اتصال شبکه برقرار باشد  

---

# مرحله 1 — تنظیم DNS

برای بهبود دسترسی به سرویس‌های داخلی، DNS سرور را روی DNSهای الوند تنظیم کنید.

DNS های الوند:

```
92.114.50.101
92.114.50.102
```

برای اطلاعات بیشتر:

```
https://dns.alvand.li
```

### ویرایش تنظیمات DNS

فایل زیر را باز کنید:

```
sudo nano /etc/resolv.conf
```

سپس مقادیر زیر را اضافه کنید:

```
nameserver 92.114.50.101
nameserver 92.114.50.102
```

فایل را ذخیره کنید.

---

# مرحله 2 — تنظیم مخازن داخلی Ubuntu

برای استفاده از مخازن داخلی، اسکریپت آماده زیر را اجرا کنید.

```
bash <(curl -s https://bash.alvand.li/core/mirror.sh)
```

پس از اجرای اسکریپت، از شما خواسته می‌شود نوع مخزن را انتخاب کنید.

برای استفاده از مخازن داخلی:

```
1
یا
4
```

را وارد کنید.

اسکریپت به صورت خودکار مخازن مناسب را تنظیم می‌کند.

---

# مرحله 3 — بروزرسانی مخازن

پس از تنظیم DNS و مخازن، لیست پکیج‌ها را بروزرسانی کنید:

```
sudo apt update
```

اگر همه چیز درست تنظیم شده باشد، مخازن داخلی بدون مشکل بارگذاری خواهند شد.

---

# تست اتصال

برای بررسی عملکرد DNS می‌توانید دستور زیر را اجرا کنید:

```
ping google.com
```

یا:

```
nslookup ubuntu.com
```

# نکات مهم

• برخی شبکه‌ها ممکن است DNS را Override کنند  
• در صورت تغییر نکردن DNS، سرویس شبکه را ریستارت کنید  
• در صورت بروز مشکل، سرور را یک بار reboot کنید
